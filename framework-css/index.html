Objetivos da Aula

  • Compreender o que é o Bootstrap e sua importância no desenvolvimento web.
  • Construir páginas responsivas usando o sistema de grid e componentes.
  • Conhecer outros frameworks CSS e comparar suas características.

Introdução ao Bootstrap

  • Framework CSS open source criado pelo Twitter.
  • Facilita a criação de interfaces modernas e responsivas.
  • Trabalha em conjunto com HTML, CSS e JS.

Por que usar o Bootstrap?

  • Acelera o desenvolvimento: Com uma vasta coleção de componentes e estilos prontos, você não precisa programar tudo do zero.
  • Responsividade: O sistema de grid (grade) do Bootstrap permite criar layouts que se ajustam automaticamente ao dispositivo do usuário.
  • Consistência: Garante uma aparência padronizada e profissional para seu site, mesmo sem ser um especialista em design.
  • Fácil de usar: É baseado em HTML, CSS e JS, exigindo apenas um conhecimento básico dessas tecnologias para começar.

O sistema de grid do Bootstrap é baseado em 12 colunas e foi pensado para mobile‑first (você define o comportamento em telas pequenas e vai "abrindo" o layout conforme os breakpoints).

Conceitos‑chave

  • Container: área máxima do conteúdo. Use .container (largura máxima por breakpoint), .container-fluid (100% da largura) ou .container-{breakpoint} (fluido até certo ponto).
  • Row: agrupa colunas na horizontal e aplica gutters (espaços) automaticamente.
  • Column: define a largura do conteúdo. Pode ser automática (.col) ou explícita (.col-6, .col-md-4, etc.).

Exemplos

1) Container + linha + colunas (base)

<div class="container">
  <div class="row">
    <div class="col">A</div>
    <div class="col">B</div>
    <div class="col">C</div>
  </div>
</div>

.col divide o espaço restante igualmente entre as colunas.

2) Larguras fixas e responsivas

<div class="container">
  <div class="row">
    <div class="col-12 col-md-8">Conteúdo principal</div>
    <div class="col-12 col-md-4">Barra lateral</div>
  </div>
</div>

Em telas pequenas: stack (12/12). A partir de md: 8/12 + 4/12.

3) Gutters (espaçamentos entre colunas)

Use g-* (ambos eixos), gx-* (horizontal) e gy-* (vertical).
Valores vão de 0 a 5.

<div class="container">
  <div class="row g-3">
    <div class="col-6 col-lg-3"><div class="p-3 bg-light">1</div></div>
    <div class="col-6 col-lg-3"><div class="p-3 bg-light">2</div></div>
    <div class="col-6 col-lg-3"><div class="p-3 bg-light">3</div></div>
    <div class="col-6 col-lg-3"><div class="p-3 bg-light">4</div></div>
  </div>
</div>

4) Alinhamento vertical e horizontal

Na row: align-items-* (vertical), justify-content-* (horizontal). Em cada col: align-self-*.

<div class="container border border-danger vh-100 border-3 d-flex">
  <div  class="row align-items-center justify-content-between border border-2 border-info flex-fill">
        <div class="col-4 align-self-end">
                <div class="bg-light p-3 text-center">Esquerda</div>
        </div>
        <div class="col-4">
                <div class="bg-light p-3 text-center">Centro</div>
        </div>
  </div>
</div>

5) Ordenação e offsets

Reordene visualmente com order-* e crie recuos com offset-*.

<div class="container">
  <div class="row">
    <div class="col-12 col-md-4 order-md-2">Coluna 2 (aparece depois em md+)</div>
        <div class="col-12 col-md-4 order-md-1">Coluna 1</div>
        <div class="col-12 col-md-4 order-md-0 offset-md-2">Coluna 3</div>
  </div>
</div>

Dica: order-0, order-1, order-2... e variantes por breakpoint (order-md-1).

Para recuos: offset-md-2 desloca a coluna em 2 colunas à direita no md.

6) Colunas automáticas e com conteúdo

.col-auto faz a coluna "abraçar" o conteúdo, enquanto as demais ocupam o restante.

<div class="container-fluid border border-4 border-danger vh-100">
  <div class="row border border-3 border-dark">
    <div class="col-auto">
        <div class="bg-light p-2">Label:</div>
    </div>
    <div class="col">
      <input class="form-control" placeholder="campo amplo" />
    </div>
  </div>
</div>

7) Aninhamento (rows dentro de colunas)

<div class="container-fluid border border-4 border-danger vh-100">
  <div class="row border border-3 border-dark m-1">
    <div class="col border border-3 border-info m-1">
      <div        class="row gx-3 border border-3 border-warning m-1">
        <div class="col-6"><div class="p-2 bg-light">A1</div></div>
        <div class="col-6"><div class="p-2 bg-light">A2</div></div>
      </div>
    </div>
    <div class="col p-0 border border-3 border-success m-1">
      <div class="p-0 bg-secondary">B</div>
    </div>
  </div>
</div>

8) Containers: fixo vs fluido

  • Fixo (.container): largura máxima muda por breakpoint (legível em telas grandes).
  • Fluido (.container-fluid): sempre 100% (ótimo para apps que ocupam toda a largura).
<div class="container-fluid">
  <div class="row g-4">
    <div class="col-12 col-md-6 col-xxl-3"><div class="p-3 bg-light">Card 1</div></div>
    <div class="col-12 col-md-6 col-xxl-3"><div class="p-3 bg-light">Card 2</div></div>
    <div class="col-12 col-md-6 col-xxl-3"><div class="p-3 bg-light">Card 3</div></div>
    <div class="col-12 col-md-6 col-xxl-3"><div class="p-3 bg-light">Card 4</div></div>
  </div>
</div>

Boas práticas

  • Pense primeiro no mobile (comece com .col-12 e vá refinando nos breakpoints).
  • Prefira utilitários (g-*, p-*, m-*, text-*, d-flex) antes de criar CSS próprio.
  • Use aninhamento para composições complexas e gx/gy para controlar espaços independentes.
  • Evite linhas com soma de colunas > 12 (quebra de linha inesperada).
  • Teste em tamanhos reais (DevTools → Responsive Design Mode).

Instruções de Entrega

  • Todos os exercícios devem ser entregues em um repositório GitHub.
  • Cada exercício deve estar em uma pasta separada com README explicando o que foi feito.

Critérios de avaliação

  • Layout Responsivo: grade, breakpoints, sem overflow.
  • Usabilidade e acessibilidade: foco, contraste, navegação por teclado.
  • Qualidade visual: hierarquia, espaçamentos, consistência de componentes.
  • Evidências e justificativas: prints, medições, razões de escolha.
  • Processo: commits incrementais com mensagens claras.

Regras gerais anti‑dependência de IA

Entrega em GitHub com commits incrementais (mensagens descritivas do que foi pensado/testado).

Em cada exercício, inclua no README:

(a) rascunho/low‑fi (foto ou link)

(b) evidência de teste (print do DevTools)

(c) decisões descartadas (o que você tentou e por que não manteve)

Não use outras bibliotecas: apenas Bootstrap 5 e HTML

Observação ao aluno

Usar IA como apoio é permitido para dúvidas conceituais, mas a nota depende das suas decisões e evidências. Copiar trechos sem entender levará a um layout incoerente e a perda de pontos.
